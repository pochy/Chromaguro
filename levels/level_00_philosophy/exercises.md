# Level 0 Exercises

## 1. 検索対象を決める

自分が作りたい RAG アプリを 1 つ選び、次を埋めてください。

```text
対象ユーザー:
ユーザーが聞きたいこと:
検索対象の文書:
回答に必ず表示したい source 情報:
検索してはいけない文書:
```

## 2. 「近い」と「答え」を分ける

`data/sample_questions.md` から質問を 3 つ選びます。それぞれについて、以下を書いてください。

```text
意味的に近そうな文書:
本当に答えになる文書:
metadata で絞るべき条件:
keyword / full-text が必要な語:
```

## 3. Collection 設計を試す

次の 2 案を比較してください。

```text
A: すべての文書を 1 collection に入れ、metadata で絞る
B: tenant / 文書種類 / embedding model ごとに collection を分ける
```

比較する観点:

```text
検索品質
権限管理
再 index
評価のしやすさ
運用の複雑さ
```

## 4. SQLite と Chroma の責任分界を書く

自分のアプリで、次をどちらに置くか決めてください。

```text
ユーザーアカウント
権限
原本文書の保存場所
chunked document
embedding
検索ログ
評価データ
```

## 提出物

次の 3 つを書ければ完了です。

```text
1. 自分が作りたい RAG アプリの対象ユーザー
2. 検索対象にする文書
3. 「近い文書」と「答えになる文書」の違いの説明
```
